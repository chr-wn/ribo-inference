# Pipeline Documentation

This directory contains the core scripts for the RNA-Ligand Binding Affinity Prediction pipeline.

## Key Scripts

### 1. Orchestrators
- **`run_full_pipeline.sh`**: 
  - **Purpose**: Runs the entire training workflow (Feature Generation -> Consolidation -> Training).
  - **Outputs**: 
    - Features: `data/processed/`
    - Pickles: `data/consolidated/`
    - Models: `models/inference_ensemble/`
  - **Dev Mode**: Use `--dev` for a fast 1-epoch test run (outputs to `models/dev_test`).

- **`run_inference.sh`**: 
  - **Purpose**: Runs predictions on a new dataset.
  - **Inputs**: A CSV file with RNA and Molecule columns.
  - **Outputs**: A CSV file with predictions (`pred_pKd`, `uncertainty_sigma`).
  - **Note**: Requires normalization parameters (`--norm-mean`, `--norm-std`) to output correct pKd values.

- **`setup.sh`**: 
  - **Purpose**: Manages Conda environments.
  - **Env Cache**: To speed up Google Colab sessions (which reset often), this script caches created environments as `.tar.gz` files in `inference/env_cache` (path customizable). It restores from cache instead of reinstalling when possible.

### 2. Python Modules
- **`generate_all_features.py`**: 
  - Orchestrates detailed feature generation.
  - Calls sub-scripts in `rna/` (RiNALMo, MXFold2, MMseqs2) and `mol/` (UniMol, RDKit).
- **`consolidate_features.py`**: 
  - Compiles thousands of individual feature files into optimized `.pkl` dictionaries for fast loading during training/inference.
- **`train_inference_model.py`**: 
  - Trains an ensemble (default 5 models) of RMPred.
- **`predict_inference.py`**: 
  - Loads models and consolidated data to generate predictions.

## Data & Artifacts

### Default Datasets
- **`data/union_dataset_final.csv`**: The master dataset used for training. Contains RNA sequences, SMILES, and ground truth `pKd` (affinity) labels.
- **Redundant Files**: 
  - `unique_rnas.csv` and `unique_molecules.csv` are generated automatically from the union dataset during pipeline runs. You do not need to manage these manually.
  - `feature_manifest.json` (deprecated) has been removed.

### Output Directories
- **`data/processed/`**: Raw feature files (numpy arrays, JSONs) generated by `generate_all_features.py`.
- **`data/consolidated/`**: Combined pickle files (`rna_embed.pkl`, `mole_graph.pkl`, etc.) used by the data loader.
- **`results/`**: Recommended directory for inference outputs.

## Normalization (Critical)

The model predicts **Z-scores** of the binding affinity (pKd), not raw pKd values.
- **Why?** Neural networks train more stable when targets have zero mean and unit variance.
- **During Training**: The pipeline calculates mean/std of the training labels and normalizes targets: `(pKd - mean) / std`.
- **During Inference**: The model outputs Z-scores. To get back to pKd, we must un-normalize: `(pred * std) + mean`.
  - Since inference data usually lacks labels, we cannot calculate "inference mean/std". **You must provide the training statistics.**
  - **Default Training Stats**: Mean ≈ `4.678`, Std ≈ `2.145` (check your training logs if you retrain!).

## PSSM (Optional but Recommended)

Position-Specific Scoring Matrices (PSSMs) provide evolutionary information that improves prediction accuracy.
- **Requirement**: Requires the RNAcentral database (~100GB uncompressed).
- **Setup**:
  ```bash
  bash databases/download_rnacentral.sh
  ```
  This downloads `rnacentral_active.fasta` and builds an MMseqs2 database.
- **Fallback**: If the database is missing, the pipeline silently falls back to generating PSSMs from the sequence itself (one-hot equivalent), which is faster but less informative.
